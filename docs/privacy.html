🍷 Politique de confidentialité

Application Cavea · Dernière mise à jour : avril 2026

En résumé : Cavea ne collecte aucune donnée personnelle. Vos données restent sur votre appareil ou dans votre propre espace Google Drive. Nous n'avons accès à rien.

1. Qui sommes-nous ?

Cavea est une application personnelle de gestion de cave à vin développée par Alain Benard. L'application est disponible sur Windows et Android. Code source disponible sur GitHub.

2. Données collectées

Cavea ne collecte aucune donnée personnelle. Il n'y a pas de compte utilisateur, pas de serveur distant, pas d'analytique, pas de publicité, pas de cookies.

Les seules données traitées par l'application sont :

  • Votre cave à vin (bouteilles, millésimes, emplacements…) — stockée dans un fichier SQLite local (cave.db) sur votre appareil.
  • Vos préférences d'application (chemin du fichier, valeurs par défaut) — stockées localement dans les préférences système de votre appareil (SharedPreferences).

3. Synchronisation Google Drive (optionnelle)

Si vous activez la synchronisation Google Drive, l'application accède à votre propre espace de stockage Google Drive via l'API officielle Google, dans un dossier dédié (Cavea/). Elle y dépose et récupère uniquement votre fichier cave.db.

Ce que cela implique :

  • Votre token d'authentification Google est stocké localement sur votre appareil dans un espace sécurisé (Keychain sur Android, Credential Store sur Windows). Il n'est jamais transmis à un tiers autre que Google.
  • L'application utilise le scope drive.file : elle ne peut accéder qu'aux fichiers qu'elle a elle-même créés dans votre Drive (le dossier Cavea/, visible dans votre interface Google Drive). Elle ne peut ni lire ni modifier le reste de votre Drive.
  • Aucun fichier de votre Drive n'est lu ou modifié par le développeur.

La synchronisation est entièrement sous votre contrôle : vous pouvez la désactiver à tout moment depuis les Paramètres de l'application.

4. Partage de données

Aucune donnée n'est partagée avec des tiers. Il n'y a pas de serveur backend, pas de base de données distante, pas de service d'analyse. Le développeur n'a techniquement aucun accès à vos données.

5. Sécurité

Le fichier cave.db est stocké en clair (non chiffré) sur votre appareil. C'est un choix délibéré pour une application d'usage personnel. Si votre appareil est protégé par un code PIN ou un mot de passe, vos données sont protégées au même niveau que tous les autres fichiers de l'appareil.

6. Droits et contact

Comme Cavea ne stocke aucune donnée côté serveur, il n'y a pas de données à consulter, corriger ou supprimer auprès du développeur. Toutes vos données sont sous votre contrôle direct sur votre appareil.

Pour toute question : GitHub Issues.

7. Modifications

Cette politique peut être mise à jour lors de nouvelles versions de l'application. La date en haut de page indique la version en vigueur.
